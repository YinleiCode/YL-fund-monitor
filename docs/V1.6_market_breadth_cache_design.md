# V1.6 赚钱效应快照设计

本文描述已接入的派生脚本方案，不改变交易规则。

## 目标

只读派生脚本 `scripts/build_market_breadth_cache.py` 在收盘后生成当日真实赚钱效应快照，供 `build_market_daily.py` 读取。

拟输出：

- `output/market_breadth/market_breadth_YYYYMMDD.csv`
- `output/market_breadth/market_breadth_latest.csv`

## 字段

| 字段 | 中文含义 | 当前状态 | 未来建议来源 |
|---|---|---|---|
| `advance_count` | 涨家数 | 已由 `market_breadth` 写入，失败为空 | 全市场 spot 涨跌幅统计 |
| `decline_count` | 跌家数 | 已由 `market_breadth` 写入，失败为空 | 全市场 spot 涨跌幅统计 |
| `advance_decline_ratio` | 涨跌比 | 已本地派生，失败为空 | 本地计算 |
| `limit_up_count` | 涨停家数 | 已接 akshare 涨停池，失败为空 | akshare 涨停池 |
| `limit_down_count` | 跌停家数 | 已尝试 akshare 跌停池，失败为空 | akshare 跌停池 |
| `burst_count` | 炸板数 | 已接 akshare 炸板池，失败为空 | akshare 炸板池 |
| `burst_rate` | 炸板率 | 已本地派生，失败为空 | 本地计算 |
| `index_change_pct` | 上证涨跌 | 已接指数接口，失败为空；真实 `0.0` 保留 | 指数行情接口 |
| `total_amount` | 全市场成交额 | 已由全市场 spot 汇总，失败为空 | 全市场行情聚合 |

## 降级规则

`build_market_daily.py` 已有安全降级：

- 如果涨跌家数、涨跌停、炸板、成交额明细全部缺失，则 `market_env_verdict=数据不足`。
- 不允许只凭原始情绪分把真实赚钱效应染成强势。
- 缺字段时 `sentiment_data_status` 保持 `partial` 或 `missing`。

`market_breadth` 接入后仍保持：

- 当日文件缺失：不 fallback 历史文件。
- 文件日期不等于 `report_date`：拒绝读取。
- 字段不完整：只填可用字段，`sentiment_data_status=partial`。
- 数据源失败：保留空字段，显示“数据不足”。

## 不做

- 不修改 `run.py`。
- 不修改 `trade_review.py` 买入判定。
- 不让资金条件层变成硬拦截。
- 不使用旧行情文件冒充当日数据。
