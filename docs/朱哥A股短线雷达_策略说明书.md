## 一、系统概述

朱哥A股短线雷达是一套 A 股短线辅助选股系统，每天盘前自动从全市场筛选短线候选票，开盘后确认买入信号，收盘后自动复盘，目标是验证"量化选股 + 纪律执行"能否持续产生正收益。

系统每天并行运行两个独立模式，互不干扰：

| 模式 | 运行时间 | 逻辑 |
|------|---------|------|
| **全A模式**（full）| 08:50 | 从全A 5200+ 只股票里选前3只 |
| **主题龙头模式**（theme_auto）| 08:55 | 从当前热门主题板块里选前3只 |

两个模式都只是"候选推荐"，都需要经过统一的 9:36 买入确认才算模拟买入。

## 二、全A模式（full）：怎么选股

### 筛选流程

```
全A 5200+ 只
  → 硬排除（ST、停牌、涨跌幅异常）
  → 粗筛（成交额、换手率、价格基本门槛）
  → 拉取历史K线
  → 计算技术指标
  → 四维打分（人气 + 技术 + 空间 + 风险）
  → 按总分排名
  → 取前3只推送
```

### 四维评分说明

| 维度 | 含义 | 高分意味着 |
|------|------|----------|
| **人气分** | 资金关注度，看成交额、换手率、量能 | 有资金主动参与，市场在关注这只票 |
| **技术分** | 短线趋势强度，看均线结构、MACD、量价 | 趋势向上，形态健康 |
| **空间分** | 上涨空间，看距近期高点距离、流通盘大小 | 没有严重透支，还有上涨余地 |
| **风险分** | 风险控制分，ST、连续大涨、量价背离等会扣分 | 风险可控，没有明显隐患 |

**总分** = 四项加权综合，最高约100分。总分越高越优先推荐，但不代表一定买入。

### 限制

全A模式不区分主题，只看个股自身指标，不会特别追热门板块。好处是更客观，坏处是可能在主题行情中捕捉力度不如主题龙头模式。

## 三、主题龙头模式（theme_auto）：怎么选股

### 核心逻辑

主题行情是A股短线的主要驱动力之一。主题龙头模式的思路是：先找到当前最热的主题，再从这些主题里选最强的候选股。

### 当前跟踪的主题

PCB、CPO、光模块、商业航天、机器人、算电协同、半导体先进封装、电力、液冷、算力、**先进封测OSAT**

**先进封测OSAT** 于 2026-05-26 加入，关键词覆盖：封测、封装测试、先进封装、OSAT、Chiplet、Bumping、晶圆级封装、倒装、FC、2.5D封装、3D封装、SiP、测试、晶圆测试、成品测试。方向参考：长电科技（600584）、通富微电（002156）、华天科技（002185）等，不手动锁定股票池，由关键词匹配板块自动筛选。

### 筛选流程

```
预设主题关键词列表
  → 匹配东方财富概念板块/行业板块
  → 获取各板块今日行情（涨跌、成交额、上涨家数）
  → 计算主题强度
  → 找强主题的成分股
  → 用原系统四维评分打分
  → 计算 theme_auto_score = 原总分 × 0.8 + 主题加分
  → 按 theme_auto_score 排名
  → 取前3只推送
```

### theme_auto_score 公式

```
theme_auto_score = 原系统总分 × 0.8 + 主题加分
```

- **原系统总分 × 0.8**：个股自身质量仍是主体，降权后让出空间给主题加分
- **主题加分**：最高20分，根据主题强度给予额外奖励，让强主题里的龙头候选排名更靠前
- **0.8权重是当前实验值**，跑满一个月后再根据复盘数据决定是否调整

### 主题强度

主题强度 = 该板块今日综合活跃度（0-100），主要看板块涨幅、成交额、上涨家数占比。主题强度越高，说明这个方向今天越活跃。

主题强度高≠直接买入，仍需通过9:36确认。

### 限制

- 主题来源依赖东方财富板块数据，数据源不稳定时会降级使用磁盘缓存（最近7日内）
- 板块成分股匹配是近似的，部分股票可能被错误纳入或遗漏
- 当前主题列表是固定的，市场热点快速轮转时可能覆盖不全

## 四、大盘情绪分

### 当前版本（简化版）

大盘情绪分衡量当天整体市场环境，影响是否允许模拟买入。

**当前仅使用两个指标：**
1. 上证指数涨跌幅
2. 全市场成交额

| 情绪分 | 含义 | 对买入的影响 |
|-------|------|-----------|
| 0–4 | 市场偏弱 | 买入条件不满足，原则上不模拟买入 |
| 5–7 | 中性偏稳 | 允许模拟买入，但保持谨慎 |
| 8–10 | 市场活跃 | 正常模拟买入 |

### 当前不包含的指标

当前版本暂不包含以下专业短线情绪指标：涨停家数、跌停家数、炸板率、连板高度、昨日涨停溢价、红盘家数。

这些指标属于后续研究方向，当前不改。

## 五、9:36买入确认：为什么是这个时间

### 为什么不是9:30

9:30开盘后的前几分钟是集合竞价后的"混乱期"：
- 大单挂单撤单快
- 价格波动剧烈
- 高开低走的票很多
- 资金试盘、博弈激烈，不适合直接判断

### 为什么是9:36（约5分钟后确认）

9:36（开盘后约5-6分钟）是一个折中时间点：
- 市场已经初步稳定
- 能看出开盘承接是否真实
- 过滤掉高开低走的情况
- 如果主力想拉，通常这时候已经开始拉
- 不太早（避免混乱），不太晚（还能低位参与）

这不是一个"神奇时间点"，而是经验性的短线确认窗口。

## 六、买入条件（V1.6：复盘计划层 + 资金条件层（观察模式）+ 9:36 技术确认层）

### 6.0 核心原则

> **推荐 ≠ 买入。** 买入必须同时满足四点：① 有逻辑　② 有资金　③ 有买点　④ 风险可控。  
> 9:36 技术确认层切换日期：`config.yaml → buy_rules.version_effective_from = 20260527`。  
> 早于此日的 `report_date` 沿用历史口径；当日及之后启用当前确认层，已落盘的买入确认不重算。

### 6.1 复盘计划层预筛（不达标不进入 9:36 确认）

| 模式 | 门槛 | 原因码 | 中文 |
|------|------|--------|------|
| theme_auto | 主题强度 ≥ 50 | `theme_strength_too_low` | 主题强度不足，暂不买入 |
| full | 总分 ≥ 78 且 人气分 ≥ 22 且 技术分 ≥ 20 | `full_score_not_strong_enough` | 全A模式分数或人气技术不够强，只观察不买入 |

预闸未通过的票直接落到 `未买入观察样本`，不抓 9:36 实时行情、不写入 `buy_price` / `stop_price`。

### 6.2 开盘涨幅三段分级

| 区间 | 处理 | 原因码 | 中文 |
|------|------|--------|------|
| 开盘涨幅 < -3% | **主因否决** | `open_change_too_low_hard` | 开盘跌幅超过3%，明显弱开，直接放弃 |
| -3% ≤ 开盘涨幅 < -1% | **辅助提示，不否决** | `open_change_weak_watch` | 低开超过1%，开盘偏弱，但不单独否决 |
| 开盘涨幅 > +4% | **主因否决** | `open_change_too_high` | 开盘涨幅超过4%，高开过多 |
| 其余（-1% ~ +4%） | 通过 | — | — |

> 低开 -1%~-3% 的票，只要 9:36 能站回开盘价并站上 5日线，允许模拟买入。

### 6.3 9:36 共有标准（四条全部满足）

| # | 条件 |
|---|------|
| 1 | 大盘情绪分 ≥ 5 |
| 2 | 9:36 价 ≥ 开盘价 |
| 3 | 9:36 价 ≥ 5日均线 |
| 4 | 非一字涨停（可成交） |

### 6.4 买入时报告必填「四因」

| 维度 | 模板 |
|------|------|
| **逻辑** | theme_auto：属于强主题（强度 ≥50）；full：总分 ≥78 的强势票 |
| **资金** | 人气分 ≥22，配合昨日成交额体现资金参与度 |
| **买点** | 9:36 价同时站上开盘价和 5日均线，承接和趋势均确立 |
| **风险** | 开盘涨幅落在 [-3%, +4%]，非一字涨停可成交 |

### 6.5 不买入时报告必填「主因 / 辅助」

| 类别 | 包含的原因码 |
|------|-------------|
| **主因** | `market_sentiment_below_5`、`theme_strength_too_low`、`full_score_not_strong_enough`、`open_change_too_high`、`open_change_too_low_hard`、`price_below_open`、`price_below_ma5`、`unable_to_buy_limit_up` |
| **辅助** | `open_change_weak_watch`（只提示不否决） |

未触发买入时仍纳入"未买入跟踪"，T+1 后观察是否错过大涨。

## 六·5、10:00 二次确认观察（V1.6 复盘观察项 — 不改正式买入规则）

> 「观察通过 ≠ 模拟买入」：second_check 通过的票**不写 `buy_signal_0935=true`，不写 `buy_price`，不计入正式收益、止损、T+1 复盘**。  
> 仅用于月底分析：9:36 没买、但 10:00 重新走强的票，后续表现如何。

### 6·5.1 适用对象

只处理当日已完成 9:36 检查（`price_0935` 有值）、`buy_signal_0935=false` 且失败原因属于「可观察白名单」、且当日尚未观察过的样本。

### 6·5.2 失败原因白/黑名单

| 类别 | 包含 |
|------|------|
| 可二次观察 | `price_below_open`、`price_below_ma5`、`open_change_weak_watch` |
| 不可二次观察 | `market_sentiment_below_5`、`theme_strength_too_low`、`full_score_not_strong_enough`、`open_change_too_low_hard`、`open_change_too_high`、`unable_to_buy_limit_up` |

### 6·5.3 二次观察通过条件（四条全部满足）

| # | 条件 |
|---|------|
| 1 | 10:00 价 ≥ 开盘价 |
| 2 | 10:00 价 ≥ 5日均线 |
| 3 | 10:00 价 > 9:36 价（严格大于） |

### 6·5.4 新增字段

### 6·5.5 与正式买入完全解耦

## 七、买入价、滑点、止损

### 三个价格

| 价格 | 计算方式 | 含义 |
|------|---------|------|
| **模拟买入价** | 9:36确认时的实时价格 | 信号触发价 |
| **滑点后买入价** | 模拟买入价 × 1.001 | 模拟真实成交成本（报价延迟、买卖价差） |
| **止损价** | 滑点后买入价 × 0.97 | 亏损到-3%时的止损线 |

### 为什么要-3%止损

短线交易最大的风险不是输一次，而是小亏不止损最终拖成大亏。

-3%止损的逻辑：
- 设置一个明确的出局线，超过就不再等待
- 把单笔最大亏损控制在可接受范围内
- 长期来看，盈亏比比胜率更重要——只要平均盈利大于平均亏损，长期就有正收益

止损不是保证赚钱，是控制单笔风险。

## 八、T+1复盘规则

### 为什么是T+1

A股实行T+1制度，当天买入当天不能卖出。所以系统统一在T+1日获取全天数据进行复盘。

### 模拟卖出逻辑（按优先级）

```
情形1：T+1开盘价 ≤ 止损价
  → 按T+1开盘价模拟卖出（隔夜跳空低开，直接开盘卖）

情形2：T+1最低价 ≤ 止损价（但开盘没有跳空到止损线以下）
  → 按止损价模拟卖出（盘中触及止损线，按止损价卖）

情形3：全天最低价都没有跌破止损价
  → 按T+1收盘价模拟卖出（持有到收盘卖出）
```

**当前版本不设置主动止盈。** 不管涨多少，如果没有触发止损，都持有到收盘。这是当前实验规则，V1.6计划研究止盈规则。

## 九、复盘统计指标说明

### 基础结果指标

| 指标 | 含义 | 怎么看 |
|------|------|-------|
| **次日最高收益** | (T+1最高价 - 滑点后买入价) / 滑点后买入价 | 买入后有没有给过赚钱机会，不代表能真的卖到最高 |
| **次日收盘收益** | (T+1收盘价 - 滑点后买入价) / 滑点后买入价 | 如果持有到收盘的收益 |
| **最大回撤** | (T+1最低价 - 滑点后买入价) / 滑点后买入价 | 买入后最多亏过多少，负数越大说明最大浮亏越深 |
| **模拟交易收益** | 按当前卖出规则实际结算的收益 | 触发止损用止损价，否则用收盘价 |

### 成功率指标

| 指标 | 定义 | 怎么看 |
|------|------|-------|
| **冲高3%比例** | 次日最高收益 ≥ 3% 的比例 | 有多少票给过明显的短线获利机会 |
| **冲高5%比例** | 次日最高收益 ≥ 5% 的比例 | 有多少票走出了强势行情 |
| **收盘胜率** | 次日收盘价高于买入价的比例 | 持有到收盘能否盈利 |
| **止损率** | 触发-3%止损的比例 | 被止损的频率，越低越好 |
| **风险调整后成功率** | 次日最高收益≥3% **且** 最大回撤没有超过-3% | 最严格的成功定义：有冲高机会，且没有先被止损出局 |

### 路径不确定

如果同一天既冲高到3%以上，又跌破了-3%止损线，只看日线无法判断实际的交易路径（是先冲高再跌，还是先跌触止损再反弹）。这种情况标记为"路径不确定"，需要人工查看分时图确认。

## 十一、未买入跟踪

未买入票不影响模拟收益统计。它存在的目的是回答一个问题：

**"被系统放弃的票，后来涨了多少？"**

如果大量未买入票次日涨幅超过3%，说明买入条件可能过严，漏掉了太多机会。

当前判断标准：未买入票次日最高涨幅（以推荐收盘价为参考基准）≥ 3%，标记为"可能错过大涨"。这个数据不计入系统盈亏，只供参考。

| 数据依赖 | 依赖东方财富/新浪实时行情，数据源不稳定时可能缺数据；fallback 到过期缓存的当日只观察不写入复盘 |
| 主题列表固定 | theme_auto 当前主题是手工维护，不能自动发现新热点 |
| 大盘情绪简化 | 只用上证指数+成交额，不包含涨停家数、炸板率等专业情绪指标 |
| 无主动止盈 | 持有到收盘或触止损，不设价格目标 |
| T+1卖出近似 | 只用开盘/最低/收盘三个价位，不模拟分钟级卖出时机 |
| 路径不确定 | 日线数据无法判断分时走势，部分交易需人工确认 |
| 无仓位管理 | 每笔模拟买入都按相同仓位，不区分仓位大小 |
| 无复利模拟 | 收益率独立计算，不累计账户净值 |
