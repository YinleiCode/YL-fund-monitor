## 2026-05-29：修复 `_score_dist_60d` 反向惩罚创新高票

**文件：** `scorer.py`

**问题：**
`_score_dist_60d()` 函数计算距 60 日高点的得分时，给创新高股票打了低分：
- `dist_pct=0`（刚好在 60 日高点）→ 3 分
- `dist_pct=-10%`（距高点回落 10%）→ 10 分（最高分）

这导致打分模型系统性回避了突破形态的强势股，与短线追强策略的定位矛盾。

**修改：**
- 创新高或等于 60 日高点（`dist_pct >= 0`）：改为 10 分（满分）
- 距高点越远得分递减：
  ```
  0% 以下（创新高）→ 10
  0%~-5%（近高点） → 10~9
  -5%~-10%        → 9~8
  -10%~-15%       → 8~6
  -15%~-20%       → 6~4
  -20%~-25%       → 4~3
  -25%~-35%       → 3~1
  < -35%          → 1~0
  ```

**影响范围：**
- 更倾向于选出创新高、突破平台的强势股
- 远离高点的弱势股得分下降
- 不影响筛选条件（`filters.py`），不影响买入规则（`trade_review.py`）
- 不影响风险控制扣分（`score_risk`）

**测试验证：**
- `dist_pct=0` → 10.0（原 3.0）
- `dist_pct=-5%` → 9.0
- `dist_pct=-10%` → 8.0
- `dist_pct=-20%` → 4.0
- `dist_pct=-50%` → 0.0
